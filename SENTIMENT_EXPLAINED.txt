┌──────────────────────────────────────────────────────────────────────────────┐
│ 📋 HOW IT DECIDES: POSITIVE vs NEGATIVE vs NEUTRAL                           │
└──────────────────────────────────────────────────────────────────────────────┘

    INPUT TEXT
        ↓
    ┌───────────────────────────────┐
    │ 1. Convert to lowercase       │
    └───────────────┬───────────────┘
                    ↓
    ┌───────────────────────────────┐
    │ 2. Count keyword matches:     │
    │    ✅ pos_hits = 0            │
    │    ❌ neg_hits = 0            │
    └───────────────┬───────────────┘
                    ↓
    ┌───────────────────────────────────────────────────────────┐
    │ 3. Decision Logic:                                        │
    │                                                           │
    │    IF neg_hits > pos_hits AND neg_hits > 0:              │
    │       ↳ ❌ NEGATIVE                                       │
    │                                                           │
    │    ELSE IF pos_hits > neg_hits AND pos_hits > 0:         │
    │       ↳ ✅ POSITIVE                                       │
    │                                                           │
    │    ELSE check special phrases:                           │
    │       • "helped a bit" → ✅ POSITIVE                      │
    │       • "some improvement" → ✅ POSITIVE                  │
    │       • "didn't help" → ❌ NEGATIVE                       │
    │       • "no improvement" → ❌ NEGATIVE                    │
    │                                                           │
    │    ELSE:                                                 │
    │       ↳ ○ NEUTRAL (default)                              │
    └───────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│ 💡 WHY THIS APPROACH?                                                        │
└──────────────────────────────────────────────────────────────────────────────┘

✅ PROS:
• Fast - no ML model loading
• Simple - easy to understand and debug
• Transparent - you know exactly why it chose a sentiment
• Works well for clear medical outcomes

❌ CONS:
• Can't handle complex/sarcastic language
• Ties (equal pos/neg) default to neutral
• Misses context (e.g., "not bad" is positive but has "not" + "bad")
